# Plan Feature Command

Before coding, create a feature plan.

## Required Sections

### 1. User-Facing Behavior

Explain what the user will see or do.

### 2. Backend Changes

List new or changed:

- API routes
- Services
- Schemas
- Models
- Background jobs if any

### 3. Frontend Changes

List new or changed:

- Pages
- Components
- Hooks
- API clients
- Loading states
- Error states

### 4. Database Changes

List schema changes.

### 5. AI/RAG Changes

List prompt, retrieval, scoring, or citation changes.

### 6. Tests

List tests that should be added or updated.

### 7. Risks

Call out unclear assumptions, security concerns, or possible demo issues.

## Rule

Do not overbuild. Prefer the smallest useful version that improves the MVP.
