# GrantPilot Demo Script

## Demo Goal

Show that GrantPilot can help a small nonprofit quickly understand whether it is ready to apply for a specific grant.

## Demo Scenario

Organization:

BrightPath Youth Foundation

Mission:

BrightPath provides after-school STEM mentoring and academic support to low-income middle school students in Columbus, Ohio.

Grant:

Community STEM Access Fund

Goal:

Determine whether BrightPath is eligible, identify missing documents, and draft first-pass grant answers.

## Demo Documents

Upload these sample documents:

1. Mission Statement
2. Program Description
3. Annual Budget
4. Impact Summary
5. Grant Opportunity

Optional missing documents:

- IRS determination letter
- Board list
- Matching funds letter

These should intentionally be missing so the app can show useful risk flags.

## Demo Flow

### Step 1: Sign In

Show the user logging into GrantPilot.

Talking point:

"GrantPilot is built for small nonprofit teams that may not have a dedicated grant writer."

### Step 2: Create Organization

Create or open BrightPath Youth Foundation.

Talking point:

"The system starts with a basic organization profile, but most evidence comes from uploaded documents."

### Step 3: Upload Documents

Upload nonprofit documents and the grant opportunity.

Talking point:

"GrantPilot parses the documents, chunks the text, stores metadata like page numbers, and prepares the content for retrieval."

### Step 4: Run Analysis

Click "Analyze Grant Fit."

Talking point:

"The system extracts grant requirements, then compares each one against evidence from the nonprofit's uploaded documents."

### Step 5: Show Scores

Show:

- Eligibility score
- Readiness score

Talking point:

"The score is not just an AI guess. It is based on requirement coverage, missing documents, and evidence strength."

### Step 6: Show Requirement Table

Show each requirement with:

- status
- matched evidence
- citation
- confidence

Talking point:

"Every requirement is mapped back to source evidence so the user can see why the system made its recommendation."

### Step 7: Show Missing Items

Show missing documents and weak sections.

Talking point:

"This is where the app becomes actionable. It tells the nonprofit what to fix before applying."

### Step 8: Show Draft Answers

Show generated draft answers.

Talking point:

"The draft answers are grounded in uploaded evidence and are meant to be edited, not blindly submitted."

### Step 9: Download Report

Download the readiness packet PDF.

Talking point:

"The final deliverable is a grant readiness packet that the nonprofit can review internally or hand to a grant writer."

## Demo Success Criteria

The demo should clearly show:

- Document upload
- Grant requirement extraction
- Evidence matching
- Eligibility scoring
- Missing document detection
- Draft answer generation
- PDF download
